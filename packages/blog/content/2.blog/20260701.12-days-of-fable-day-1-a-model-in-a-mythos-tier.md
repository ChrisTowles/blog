> On the first day of Fable, Anthropic gave to me:
> **a model in a Mythos tier.**

Anthropic released [Claude Fable 5](https://www.anthropic.com/news/claude-fable-5-mythos-5) today. It's the first model in the Claude 5 family, and it doesn't slot into the lineup you know — Haiku, Sonnet, Opus — it sits _above_ Opus, in a new tier Anthropic is calling **Mythos-class**.

It's July. It's 90 degrees outside. And I feel like it's Christmas morning.

So I'm leaning in. This is day one of **The 12 Days of Fable** — a Christmas-in-July series where I unwrap one gift per day for twelve days. Each post is one capability, grounded in what I'm actually doing with it that day. Sing along if you know the tune.

## What's a Mythos tier?

Two names, one model. **Claude Fable 5** is the generally available version — you and I can use it today, and it ships with additional safety measures around dual-use capabilities. **Claude Mythos 5** is the same underlying model without those measures, available only to approved organizations. The model id, if you're wiring it up yourself, is `claude-fable-5`.

The interesting part isn't the naming. It's that Anthropic decided Opus — the "top" model for two years of my muscle memory — now has something sitting above it. Every time I've recalibrated my sense of "what the big model can do," it took me weeks to actually internalize it. I don't think I'll get weeks this time.

## Day one, hour one

I did what I always do with a new model: gave it real work, not puzzles. Benchmarks tell you how a model interviews. Your own backlog tells you how it works.

My backlog item was not small. I've been meaning to migrate [`towles-tool`](https://github.com/ChrisTowles/towles-tool) — my personal CLI and Claude Code plugin, all TypeScript on Bun — to Rust. "Been meaning to" is doing heavy lifting in that sentence. My brother Patrick has been telling me to use Rust for _years_. I've been nodding politely for the same number of years.

Launch night, I pointed Fable at it.

By 9:51 PM there was a new repo with a Cargo workspace, scaffolded from [Yaak's](https://github.com/mountain-loop/yaak) MIT-licensed golden template: Tauri-free shared crates, a `clap` CLI, a React + Vite frontend, and a Tauri 2 desktop shell. By the time I gave up and went to bed, migration slices were landing as clean conventional commits — journal commands at 10:34, GitHub helpers at 11:04, token-accounting graph at 11:47.

I can prove the timeline, because everything both of us did is on disk:

```bash
$ git log --pretty='%ad %s' --date=format:'%H:%M' | tail -4
21:57 feat: Tauri 2 shell (tt-app) and React client hello world
21:55 feat: cargo workspace, tt-config, tt-exec, tt-cli (ttr) with config + doctor
21:51 docs: license, attribution to yaak (MIT), README, CLAUDE.md, migration backlog
```

That's the whole story of day one: a years-old "someday" project went from zero to compiling workspace in one evening, and the commits read like a team did it.

But what happened on night two is the actual mind-blowing part, and it deserves its own post. Tomorrow.
