> On the second day of Fable, Anthropic gave to me:
> **two nights of Rust,**
> and a model in a Mythos tier.

[Yesterday](/blog/12-days-of-fable-day-1-a-model-in-a-mythos-tier) I told you Fable scaffolded a Rust rewrite of my TypeScript CLI on launch night. Tonight it handed me a desktop app.

Let me back up, because the timeline matters and I can prove all of it.

## Patrick was right

My brother Patrick has spent years telling me to use Rust. Years. Family dinners. Text threads. That one road trip. My counterargument was never "Rust is bad" — it was "I have a working TypeScript CLI on Bun and a finite number of evenings."

That argument died on July 1st at 9:51 PM.

Fable and I scaffolded `towles-tool-rs` from [Yaak's](https://github.com/mountain-loop/yaak) golden template: a Cargo workspace with Tauri-free shared crates, a `clap` CLI, and a React + Vite frontend under a Tauri 2 shell. Then it started porting my actual tool, feature by feature, as migration slices:

```bash
$ git log --pretty='%ad %s' --date=format:'%m-%d %H:%M'
07-02 00:09 feat: port install command and doctor history/diff
07-01 23:47 feat: port graph token-accounting logic to tt-graph crate
07-01 23:04 feat: port gh commands (migration slice 2)
07-01 22:34 feat: port journal commands (migration slice 1)
07-01 21:57 feat: Tauri 2 shell (tt-app) and React client hello world
07-01 21:55 feat: cargo workspace, tt-config, tt-exec, tt-cli (ttr)
```

Journal at 10:34 PM. GitHub helpers at 11:04. Token-accounting graph before midnight. Each slice a vertical cut: port the crate, wire the CLI, keep the tests green. On a follow-up pass Fable did cleanup and helped migrate my bun-cli library too — the second pass is where a rewrite usually stalls, and it just... didn't.

If you don't believe an AI did this overnight, the receipts are grep-able. Every Claude Code session is a transcript on disk:

```bash
$ grep -rl "towles-tool-rs" ~/.claude/projects/ | head
```

Timestamps don't lie. More on log archaeology on day 6.

## Night two: AgentBoard

Here's the part that got me.

I run a tmux setup with a custom sidebar I call AgentBoard — it watches every repo slot on my machine and shows which coding agents are running, finished, or stuck. Custom theme, custom status line, years of tweaking. It's the thing I'd rebuild last, because it's the thing most shaped like _me_.

Tonight, Fable recreated it as a real desktop app. By 1:40 AM the commit log reads like a sprint review:

- `00:41` — AgentBoard core engine ported to a `tt-agentboard` crate
- `00:55` — claude-code agent watcher
- `00:58` — React UI over a mock state source
- `01:07` — Tauri bridge and repo-paths config
- `01:40` — watchers for codex, amp, and opencode; open-in-editor command

My tmux sidebar — theme and all — became a Tauri app with watchers for four different coding agents, in about an hour.

And no, before you ask: **it isn't a VS Code clone.** Every AI-built desktop app demo lately is an editor with a chat panel. This is the opposite — a dashboard for the agents doing the editing. The editor is wherever you like it; AgentBoard just tells you which of your five repo slots needs a human.

## The honest part

Not everything went perfectly, and I think the imperfection is the most useful data point.

The plan was a hard cutover: the Rust binary ships as `ttr` during migration, then takes over the `tt` name at feature parity. This morning at 8:34 AM, riding the high, I let it do the rename. By 5:50 PM I reverted it. Parity wasn't there yet, and a CLI you use forty times a day is a bad place to discover that.

That's the real shape of working with a Mythos-tier model: it compresses months into nights, and it will also cheerfully help you cut over one day too early. The judgment about _when_ is still your job.

Two nights. A Cargo workspace, six ported feature slices, a desktop app that isn't a VS Code clone, and one humbling revert.

Patrick, you were right. It only took a frontier model to get me there.
